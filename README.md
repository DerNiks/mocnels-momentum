TUGAS 4
1. AuthenticationForm merupakan sebuah form bawaan dari Django yang berguna untuk melakukan validasi kredensial dari pengguna saat ingin melakukan proses login (username dan password). Form ini akan memeriksa apakah username dan password yang dimasukkan oleh user cocok atau tidak.

Kelebihan:
Form ini sangat membantu kita untuk menangani berbagai aspek keamanan karena dapat mencegah timing attack pada saat melakukan validasi password. Timing attack merupakan salah satu serangan siber yang memanfaatkan perbedaan waktu dalam pemrosesan data atau eksekusi fungsi oleh sistem untuk mengekstrak informasi sensitif dari pengguna. Selain itu, pengguunaan AuthenticationForm dapat menghemat waktu dalam pembuatan kode karena kita tidak perlu untuk memikirkan logika validasi dari awal. Kita hanya perlu mengimport AuthenticationForm dan menggunakannya sehingga sangat mengehmeat waktu development.

Kekurangan:
AuthenticationForm hanya akan melakukan validasi pada username dan password saja, sehingga mengharuskan developer untuk menambahkan field lain pada form login jika memang memerlukan data lain seperti email dan lain-lain. Dari sisi design, developer membutuhkan waktu lebih untuk menyesuaikan tampilan dari form tersebut agar dapat sesuai dengan design yang diinginkan. Hal ini dikarenakan developer perlu untuk secara manual menambahkan style baerupa css maupun atribut html pada templatenya. AuthenticationFom juga hanya berlaku pada fitur login saja, sedangkan pada fitur register kita tidak memakai authenticationform untuk melakukan validasi.

2. Autentikasi merupakan sebuah proses untuk memvalidasi identitas dari user. Penerapan autentikasi dapat dilihat melalui proses login, dimana program seakan-akan bertanya kepada kita "Siapa Kamu?" sehingga kita sebagai user perlu untuk menginput username dan password yang sesua. Sedangkan, otorisasi merupakan sebuah proses untuk memvalidasi (memberikan maupun menolak) hak akses para pengguna yang sudah terautentikasi terhadapat sesuatu. Salah satu penerapan otorisasi adalah pengguna biasa tidak dapat melakukan penghapusan data, sedangkan seorang admin dalam melakukan hal tersebut. Dalam Django, autentikasi dapat di import dari django.contrib.auth yang didalamnya menyediakan berbagai atribut autentikasi, seperti AuthenticationForm, login(), logout(), authenticate(), dan masih banyak lagi. Sedangkan, django mengimplementasikan otorisasi melalui decorator seperti @login_required yang dipakai dalam tugas kali ini sehingga mengharuskan pengguna untuk melakukan login terlebih dahulu sebelum mendapatkan hak akses dalam website.

3. Cookies memiliki kelebihan pada jangka waktu dan beban server. Cookies dapat disimpan dalam browser dalam jangka waktu yang lama bahkan setelah browser ditutup. Hal ini bisa terjadi selama masa expired dari cookies tersebut masih belum berakhir. Sehingga ini sangat berguna bagi para user, terutama pada fitur "remember me" pada halaman login. Selain itu, server tidak perlu mengelola dan menyimpan data untuk setiap user karena data tersebut tersimpan di sisi klien sehingga dapat mengurangi penggunaana memori pada server. Kekurangan dari cookies adalah ukurannya yang sangat terbatas dan kecil, sangat rentan terhapdat serangan siber seperti CSRF dan XSS jika properti keamanannya sangat sedikit. Selain itu, pengguna juga memiliki control pada cookies, karena para pengguna dapat menonaktfkan cookies pada browser mereka jika mereka merasa tidak ingin berada dibawah pengaruh cookies tersebut. Hal tersebut tentu dapat mengganggu fungsionalitas pada website. Session memeliki kelebihan pada tingkat keamanan dan kapabilitas dimana semua data disimpan pada sisi server sehingga semua data sensitif dapat diamankan oleh server. Selain itu, session tidak memiliki batas ukuran data yang dapat disimpan. Tetapi, kelebihan tersebut juga menjadi kekurangannya, dimana session membutuhkan memori server sehingga akan menjadi sebuah masalah serius jika website tersebut memiliki banyak pengguna. Session juga tidak memiliki fitur "remember me" sehingga memaksa para pengguna untuk melakukan login kembali.

4. Pada dasarnya, penggunaan cookies tidak sepenuhnya aman, karena terdapat beberapa potensi serangan siber seperti yang saya sebutkan pada nomor 3 yaitu CRSF yang merupakan sebuah tipe vulnerability di mana sesorang bisa melakukan request dari origin yang tidak seharusnya. Salah satu mitigasi untuk menghindari CSRF pada django adalah dengan melakukan implementasi token CSRF yang disertakan pada request sehingga web dapat memastikan bahwa request yang diterima nmerupakan request yang asli. Token CSRF dibuat dengan menggunakan algoritma kriptografi yang aman sehingga tidak bisa di generate dengan sembarangan. Django juga memiliki built-in protection untuk mencegah serangan CRSF dengan menggunakan django.middleware.csrf.CsrfViewMiddleware yang aktif pada settings.py. Middleware ini merupakan komponen yang berjalan di tengah-tengah alur request Django. Jika token tidak cocok, maka django akan menolak request tersebut sehingga secara efektif melakukan blocking pada serangan CSRF. Karena penerapaan middleware ini dieksekusi sebelum view, maka perlindungan ini diterapkan secara universal pada aplikasi web. Selain itu juga terdapat serangan XSS (Cross Site Scripting) yang merupakan sebuah vulnerability di mana seseorang dapat melakukan injeksi kode JS yang akan dieksekusi oleh client sehingga memungkinkan penyerang mencuri cookies dari pengguna yang sudsah melakukan login pada sebuah website. Salah satu pencegahannya adalah selalu melakukan sanitization terhadap semua user input. Selain itu,  Content Security Policy (CSP) juga dapat digunakan untuk menghindari XSS, di mana atribut dan tag yang biasa digunakan untuk melakukan XSS dapat dibatasi penggunaannya. Django juga memiliki built-in security untuk mencegah XSS yaitu dengan menggunakan is_valid() pada template djngo, forms.py, dan views.py

1. Pada Synchronus, ketika request dikirim dari browser ke server, maka browser tersebut akan diblokir terlebih dahulu dan harus menunggu sampai server mengembalikan respons. Pada proses ini, para user tidak bisa berinteraksi dengan pagpe sehingga akan muncul page reload saja. Sedangkan, pada asynchronus, request dikirim pada background sehingga browser tetap responsif dan memungkinkan user untuk berinteraksi dengan halaman web. PAda saat respon server tiba, maka otomatis akan memperbarui bagian tertentu dari halaman tanpa memerlukan reload penuh

2. Pertama, aksi dari user seperti klik button akan ditahan oleh JS yang kemudian membuat sebuah request HTTP. Request ini akan diterima oleh Django dan akan diproses pada backendnya. Konsep AJAX pada Django adalah view tidak merender template HTML secara penuh, melainkan mengembalikan data dalam format yang lebih ringan seperti JSON dengan menggunakan JsonResponse. Data JSON ini akan dikirim kembali ke browser dimana JS akan menerima dan memprosesnya untuk memperbarui bagian spesifik dari DOM halaman tanpa memuat ulang seluruh halaman.

3. Keuntungan AJAX adalah efisiensi dan pengelaman pengguna. AJAX hanya bertukar data yang diperlukan dan bukan seluruh halaman, sehingga sangat mempercepat waktu respons secara keseluruhan. Dari sisi pengguna, keuntungan terbesar adalah non-blocking dan pembaruan yang lebih selektif sehingga pengguna tidak mengalami jeda karena tidak perlu page reload yang terjadi. Hal ini tentu membuat aplikasi web terasa lebih responsif sehingga meningkatkan kepuasan pengguna.

4.  Caranya adalah dengan pencegahan Cross-Site Request Forgery (CSRF). Pada setuap request, crsf token harus diambil dari DOM atau cookie dan wajib disertakan dalam header pada request yang dikirim oleh JS. Selain itu kita juga bisa menggunakan perlindungan bawaan dari Django seperti menggunakan AutenticationForm dan UserCreationForm karena akan secara otomatis menangani hashing password yang aman dan manajemen session. Kita juga dapat melakukan validasi ulang di sisi server menggunakan form.is_valid() untuk memastikan setiap data yang diterima memang benar dan aman sebelum disimpan ke database. 

5. AJAX memungkinkan transisi yang lebih interaktif, hal ini dikarenakan kebutuhan page reload sudah tidak digunakan. User dapat melakukan tindakan seperti login, isi form, filtering produk tanpa mengganggu konteks halaman secara keseluruhan. AJAX juga memungkinkan feedback instan melalui visual seperti loading state dan toast notification sehingga mengurangi kecemasan dari para pengguna.
